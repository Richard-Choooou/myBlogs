副屏技术选型

因为副屏需要同时支持 android 和 windows 平台，同时副屏没有与用户交互的功能，仅是展示内容，所以副屏选择使用 web 进行开发。一来可以提升开发效率，二来跨平台也更加方便。

主副屏通信

收银台主屏交互逻辑也是使用 web 进行开发，此时主副屏都是 web 平台，意味着存在两个 web 容器，导致无法直接交换数据，那么主副屏如何进行通信是目前存在的一个问题。

对此进行了如下设计：

主屏通过客户端桥接与副屏建立联系，先发送事件到客户端，再由客户端转发消息到副屏，副屏收到消息后进行相关功能的处理。对此可以使用收银台之前编写的 jsBridge 进行通信，关于零售收银台 jsBridge 通信，可在此文中详细了解，此处不做说明。
