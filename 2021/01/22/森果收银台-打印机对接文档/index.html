森果收银台-打印机对接文档

发表于 2021-01-22 更新于 2024-09-13

项目背景

在森果零售收银台项目中，需要在结算后进行小票打印。

打印机分为多种类型，普通的 58mm 小票打印机，80mm 小票打印机，标签贴纸打印机，有外置 usb 打印机，也有内置串口打印机，本文讲解这些打印机在 android 与 windows 平台的对接过程。

打印流程如下所示：

本篇主要讲解客户端层面代码设计。

项目术语

  • usb 打印机 - 指通过 usb 接口连接的打印机，一般为外置打印机，也有内置 usb 打印机，较少。
  • 串口打印机 - 指通过串口（RS232）连接的打印机，多为收银机内置打印机。
  • 58mm - 打印机宽度为 58mm 打印机，80mm 打印机同理。
  • 标签打印机 - 指打印可粘贴在商品上的标签纸打印机，一般用于打印条码等信息。

技术挑战

  • C++ 打包 Nodejs 原生扩展

完成要求

  • 对于 usb 打印机，需做到连接即可使用，无需再单独适配
  • windows 打印调用系统打印服务进行打印，降低对接成本
  • 能够识别特种打印机，供上层应用区分场景调用
  • 打印机的接入拔出有提示
  • 开启应用后能读取当前已连接的打印机，无需再进行设置

第三方类库

  • RxJava [java]
  • usb [nodejs]

打印机说明

usb 设备会在接口中提供 productId 以及 vendorId 通过这两个 ID，基本可以确认该打印是什么品牌以及什么型号，通过这两个信息，可以做到 能够识别特种打印机，供上层应用区分场景调用 这一需求。

同时 usb 设备的 interface 接口会提供一个 InterfaceClass 属性，通过这一属性，可以得知该 usb 是否为打印机设备，打印机设备 InterfaceClass 通常 为 7 , 通过这一特性，可以完成 对于 usb 打印机，需做到连接即可使用，无需再单独适配 这一需求。

系统打印

接着再说说系统打印，因为一些打印机是内置通过串口连接的打印机，对接此类打印机较为麻烦。因为大部分厂商基本上是不会提供 nodejs 的 sdk，只会提供 C++ 的 sdk，而 nodejs 是无法直接调用 c++ 代码的，需要通过 node-gpy 将 c++ 代码打包成 node 文件供 nodejs 调用，所以此类打印机对接起来较为麻烦，所以得找一个通用方案，借用 windows 系统打印是一个非常好的解决办法，因为 windows 系统打印支持任意类型打印机，没有再对接的成本。

因为篇幅限制，此文不具体讲解如何进行调用 Windows 系统打印 api，详情可以在此处了解。

结尾

至此已完成打印机的对接，回过头再看看几点完成要求

  • ✅ 对于 usb 打印机，需做到连接即可使用，无需再单独适配

    通过判断 usb 设备类型即可知道当前连接的设备是打印机

  • ✅ windows 打印调用系统打印服务进行打印，降低对接成本

  • ✅ 能够识别特种打印机，供上层应用区分场景调用

    编写 getPrinterType 函数，通过 productId 结合 vendorId 即可知道连接的打印机是什么型号

  • ✅ 打印机的接入拔出有提示

    通过订阅 usb 设备消息，告知上层应用即可完成

  • ✅ 开启应用后能读取当前已连接的打印机，无需再进行设置
