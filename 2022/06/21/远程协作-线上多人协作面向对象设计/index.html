远程协作-线上多人协作面向对象设计

发表于 2022-06-21 更新于 2024-09-13

前言

多人协作是远程协作项目最为复杂的核心逻辑，合理的面向对象设计是降低多人协作复杂度的必要条件。

需求介绍

  • 能够一对一、多对多进行音视频通话
  • 能够多人同时进行画板绘制
  • 每个人能够对各自的画笔进行定制（颜色，尺寸）
  • 每个人能够撤回或者清空自己绘制的内容，无法删除他人绘制的内容

元素分析以及面向对象设计

画板 DrawArea

web 端通常使用 canvas 完成画板功能，需要在画板上绘制箭头、圆圈、以及连线路径这三种不同的元素，箭头和圆圈的绘制与连线不同，需要一边清除之前的内容，一边进行绘制，否则就会出现如下多次绘制的情况。

因为画板的特殊性，需要保留之前绘制的内容，不能因为需要绘制箭头就将之前的内容清除掉，所以我们可以使用两个 canvas 来处理。

将两层 canvas 叠加在一起，底层 canvas 保存之前绘制内容，顶层 canvas 进行当前内容的绘制。在绘制箭头圆圈时动态清空顶层 canvas，绘制完成后，将顶层绘制的内容复制到底层，即可完成在同一个 canvas 绘制的效果。

但这样做也会有一个问题，因为画板能够支持多人同时绘制，且每个人能取消自己绘制的内容，那么如何保证在一个 canvas 中取消绘制的内容时不会将别的用户内容撤销掉呢？

答案是无法保证，每个用户绘制的内容有先后顺序，不能保证 canvas restore 时撤销的是自己上一次绘制的内容。或者使用命令模式，将用户的行为封装成命令，但这无疑增加了极大的复杂度，亦或者给每个用户一个单独的canvas进行绘制，但是这样在多人协作时，多个 canvas 也会显得不妥，实现不够优雅。

那么我们可以换个画板的实现方式，使用 SVG 呢？

svg 中的元素都是 DOM 元素，想要实现撤销效果，只需要将 dom 元素删除即可。同时 SVG 能够绘制的元素不比 canvas 少，实现画板功能绰绰有余，那么就决定使用 SVG 来实现画板吧。

绘制工具 DrawUtil

需要实现的绘制效果不是很多，分别有激光笔、圆圈、箭头、以及画笔，已知三种工具绘制的结果和绘制中的行为均不相同，例如激光笔需要有一个点确定坐标，但是鼠标抬起时需要等待三秒隐藏，圆圈和箭头工具则需要两个点来确定坐标，而画笔需要多个点位确定坐标来进行绘制。
