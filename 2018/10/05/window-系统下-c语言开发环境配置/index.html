window 系统下 c语言开发环境配置

发表于 2018-10-05 更新于 2024-09-13

前言

最近在学习c语言，在linux环境下，系统默认安装了gcc，所以开发c语言非常便利，但是在命令行窗口中写代码终究是不爽的，可是window平台配置起来有点麻烦，因为Visual Studio太过于庞大，所以不打算用，经过网上查找资料，配置好了环境，现做一次记录。

环境的安装

windows下的gcc有两个版本分别是mingw和cygwin，这里我选择的是minGW。

首先进入下载页面

http://sourceforge.net/projects/mingw/files/

下载完成后安装，默认安装到c盘就好，安装完成后需要配置环境变量。在path中添加minGW的bin路径。

完成后运行

如果弹出 MinGw installation manager 窗口，说明安装正常。此时，关闭 MinGw installation manager 窗口，否则接下来的步骤会报错。

使用命令安装gcc以及gdb，gcc 是用于编译 c 文件，gdb是用于调试应用程序的。

安装完成后就可以验证环境是否配置成功了。

新建一个c文件, 编写一段基础的c代码。

在 cmd 中输入命令 gcc test.c后会在当前目录生成a.exe，运行a.exe就可以看见控制台的输出了，此时说明环境安装成功。

开发工具

作为一个web程序员，当时是喜欢用轻量的编辑器做开发，这里我使用的是vs code，因为vs code也支持c语言的开发，更重要的是可以很方便的调试c语言。

首先在vscode中打开一个c文件，会提示安装c语言相关的支持插件。

其次对于c语言的开发调试，需要配置vscode的工程配置项。

.vscode

三个关键文件如下。

c_cpp_properties.json

首先按ctrl shfit + p组合键，调出vs code 的工具栏，输入c，找到c/cpp edit configurations，单击，此时会看见工程根目录添加了一个.vscode的文件夹以及文件夹里面有一个c_cpp_properties.json的文件，这个是c语言开发的vscode配置文件，可以在这里配置编译环境，以及头文件的目录等等信息。

launch.json

其次添加启动器，这个的功能是可以自动执行编译脚本，可以用于调试以及编译。点击vs code左侧的调试按钮，点击下拉框，添加一个配置，选择c++(windows)，这时会在.vscode中生成一个launch.json文件。
