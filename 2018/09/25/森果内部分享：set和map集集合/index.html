前言

es6新增了两种数据结构，set以及map结构，本文主要介绍其中的api以及业务场景

Weakset

只允许存储object类型的数据，根据内存地址判断元素是否是同一个，因为是弱引用，以引用计数算法为基础的垃圾回收机制不会判断这次引用是一次有效的引用。所以可以对于一些与内存泄露相关联的对象进行存储，例如dom节点，不过并没有啥用，因为只能存，不能取，业务场景极少。

Map

map与object结构相似，都是key-value形式，但是map集合的key可以是一个对象, 这一点我向用java的朋友求证过，在java中经常会用到map结构，因为java语言的特性，并没有像js这样方便，但是，在key中存储对象的行为也是极少的，所以不用过多的纠结使用object还是使用map集合，接下来简单的讲解map集合的api，后面会讲到业务场景

API

set

用于向Map集合中添加一个键值对

get

从map集合读取数据

读取的key要么是一个基本类型（String, Number, Boolean, Null, NaN, Symbol)，要么是复杂类型的引用(Object)，如果没有保留存储进map集合的复杂类型数据的引用，那么只能用迭代循环来读取到map集合中的数据

has

判断map集合中是否有某个元素，同上所述，判断元素的类型要么是基本类型，要么是复杂类型的引用

delete

删除map集合中的某个元素，判断类型依据如上所述

clear

清除map集合中所有的数据

WeakMap

于map相似，但是其中元素的key只能是object，且对于对象的引用是弱引用，所以不计入垃圾回收。API与Map相似，但是不能迭代遍历，只能使用 get()、set()、has()、delete() 四个方法。
